🧮 WP2.1 — 성적 입력 UI + 저장
Scenario WP2.1-1: 유효한 성적 입력 저장 성공

Given: 학생 계정으로 로그인되어 있으며, 해당 학기의 성적이 아직 입력되지 않았다.

When: 과목=국어, 학기=1학기, 지필=95, 수행=90으로 저장 요청을 보낸다.

Then: 200 OK와 함께 성적 레코드가 DB에 저장되고, 학생 성적 목록에 표시된다.

선행 Scenario: WP1.1-3

Scenario WP2.1-2: 점수 범위를 벗어난 입력 저장 실패

Given: 학생 계정이 로그인된 상태.

When: 과목=수학, 지필=150, 수행=90으로 성적 저장 요청을 보낸다.

Then: 400 Bad Request와 “점수는 0~100 사이여야 합니다” 오류 메시지가 반환된다.

선행 Scenario: 없음

📝 WP2.2 — 활동(세특/동아리/창체) 입력 + 저장
Scenario WP2.2-1: 활동 기록 신규 생성 성공

Given: 학생 계정으로 로그인되어 있고, 최소 1개 활동 입력 권한이 있다.

When: 활동명, 기간, 활동 내용, 유형(동아리)을 입력하고 저장한다.

Then: 200 OK와 함께 새로운 활동 레코드가 DB에 생성되고, 활동 목록에 표시된다.

선행 Scenario: WP1.1-3

Scenario WP2.2-2: 필수 필드 누락 시 저장 실패

Given: 학생 계정이 로그인된 상태.

When: 활동명 없이 활동 내용을 입력하고 저장 요청을 보낸다.

Then: 400 Bad Request와 “활동명은 필수 입력 항목입니다” 메시지가 반환된다.

선행 Scenario: 없음

📚 WP2.3 — 독서 기록 입력 + 저장
Scenario WP2.3-1: 독서 기록 추가 성공

Given: 학생 계정으로 로그인되어 있고, 독서 기록이 비어 있다.

When: 책 제목, 저자, 읽은 날짜, 간단 소감을 입력 후 저장한다.

Then: 200 OK와 함께 독서 기록이 생성되고, 독서 리스트에 표시된다.

선행 Scenario: WP1.1-3

Scenario WP2.3-2: 중복된 독서 기록 저장 시 경고

Given: 동일한 책 제목/날짜로 이미 독서 기록이 존재한다.

When: 같은 정보로 다시 독서 기록 입력을 시도한다.

Then: 409 Conflict 또는 “이미 등록된 독서 기록입니다” 경고가 반환된다(정책에 따라).

선행 Scenario: WP2.3-1

✅ WP2.4 — 학부모 승인 Flow
Scenario WP2.4-1: 학부모가 활동 기록 승인

Given: 학생이 새로운 활동 기록을 입력했고 상태가 “대기”이다.

When: 학부모가 활동 상세 화면에서 “승인” 버튼을 누른다.

Then: 활동 상태가 “승인됨” 으로 변경되고, 학생 화면에서도 승인 상태가 반영된다.

선행 Scenario: WP2.2-1, WP1.4-1

Scenario WP2.4-2: 학부모가 수정 요청

Given: 학생 활동이 “대기” 상태이고, 내용 중 일부가 부적절하다.

When: 학부모가 “수정 요청”을 선택하고 코멘트를 입력한 뒤 전송한다.

Then: 활동 상태가 “수정요청”으로 변경되고, 학생에게 알림이 전송된다.

선행 Scenario: WP2.2-1, WP1.4-1
