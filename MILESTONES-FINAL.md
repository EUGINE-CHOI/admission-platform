# 🏗️ Milestones Final (M0~M10)
**입시 정보 격차 해소형 고입 입시 플랫폼 MVP**  
**Milestone 기반 상위 전략 (v1.0)**  
_Last updated: 2025-11-29_

---

# 🎯 개요  
본 문서는 고입 입시 서비스 MVP를 구현하기 위한 **최상위 Milestones(M0~M10)**을 정의한다.  
각 Milestone은 **서비스 목적 기반의 기능 그룹**으로 구성되며,  
전체 구조는 PRD / 시스템 아키텍처 / 유저 흐름 / 데이터 흐름을 기준으로 최적화되었다.

---

# 🧩 M0 — 프로젝트 인프라 & 기본 아키텍처 구축 (Foundation Layer)

### 🎯 목적  
전체 서비스를 안정적으로 운영하기 위한 시스템 기반 구축.

### 📦 산출물  
- Monorepo 설정  
- Backend(NestJS) 초기 설정  
- DB(PostgreSQL) 연결  
- CI/CD 파이프라인  
- 기본 Auth/Config 구조

---

# 🧩 M1 — 계정/가족 계정 시스템 (Identity Layer)

### 🎯 목적  
학생·학부모·컨설턴트 역할을 명확히 구분하고,  
가족 관계를 정의하여 권한 기반 접근 구조를 완성.

### 📦 산출물  
- Email 로그인(JWT 인증)  
- Role 기반 RBAC  
- 가족 연결(초대 코드)  
- 부모의 자녀 정보 열람

---

# 🧩 M2 — 학생 데이터 입력 시스템 (Student Data Layer)

### 🎯 목적  
성적/활동/독서 데이터를 중심으로  
진단 및 AI가 활용할 핵심 Input 데이터 획득.

### 📦 산출물  
- 성적 입력(지필/수행)  
- 활동 입력(세특/동아리/창체)  
- 독서 입력  
- 부모 승인 Flow

---

# 🧩 M3 — 학교/전형/일정 자동 수집 & 관리자 승인 (Admissions Layer)

### 🎯 목적  
학교·전형·일정 데이터를 자동화된 방식으로 확보하여  
정확한 진단과 일정 관리 기반 확보.

### 📦 산출물  
- 크롤링 스크립트  
- 전형/일정 DB 구조  
- Admin 승인 Pipeline  
- 학교 상세 페이지

---

# 🧩 M4 — 고입 진단 엔진 (Diagnosis Core)

### 🎯 목적  
학생 데이터 × 전형 요건을 기반으로  
FIT / CHALLENGE / UNLIKELY 평가 제공.

### 📦 산출물  
- 진단 계산 로직  
- 학교 적합도 Top3 추천  
- 진단 리포트 페이지

---

# 🧩 M5 — AI 에이전트 Layer (AI Intelligence)

### 🎯 목적  
학생부·동아리·교과·독서·액션 플랜 등  
AI 기반 코칭 기능을 제공.

### 📦 산출물  
- AI Orchestrator  
- 생기부 문장 생성  
- 동아리/교과/독서 조언  
- 액션 플랜 생성  
- AI Feedback 시스템

---

# 🧩 M6 — 실행(Task) & 타임라인 시스템 (Execution Layer)

### 🎯 목적  
입시 준비 과정을 “실행 데이터”로 전환하여  
성장 추적과 행동 기반 코칭을 제공.

### 📦 산출물  
- Task(todo/doing/done)  
- Task 완료/피드백  
- Event Log  
- Student Timeline

---

# 🧩 M7 — 학생/학부모 대시보드 & 리포트 (UX Intelligence Layer)

### 🎯 목적  
학생·학부모가 현재 상태/진단/일정/할 일을 한눈에 확인하도록 제공.

### 📦 산출물  
- 학생 홈 대시보드  
- 학부모 3줄 요약 대시보드  
- 입시 일정 캘린더  
- PDF 리포트 생성

---

# 🧩 M8 — 컨설턴트 기능(상담·리포트) (Premium Layer)

### 🎯 목적  
프리미엄 서비스를 위한 상담/코칭 기능 구현.

### 📦 산출물  
- 컨설턴트 대시보드  
- 상담 예약/시간 관리  
- 상담 노트  
- AI 기반 상담 리포트  
- 리포트 공유 기능

---

# 🧩 M9 — 결제/구독 시스템 (Business Layer)

### 🎯 목적  
서비스의 비즈니스 모델을 완성.

### 📦 산출물  
- 결제(PG) 연동  
- 프리미엄 unlock  
- 가족 할인  
- 구독 상태 관리

---

# 🧩 M10 — Analytics & 운영 대시보드 (Growth Layer)

### 🎯 목적  
서비스 운영·성장·AI 품질 개선을 위한 분석 환경 구축.

### 📦 산출물  
- Error/Traffic Monitoring  
- KPI 대시보드(입력률/전환률/실행률)  
- AI Feedback Analyzer  
- ETL 기반 데이터 적재 구조

---

# 📌 결론  
M0~M10은  
MVP 구축을 위한 **완전하고 최적화된 상위 전략**이다.

각 Milestone은  
PRD → 아키텍처 → 유저 흐름 → 비즈니스 요구를  
모두 충족하는 완결형 구성이다.
